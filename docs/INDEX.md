## How to extend

Each subsystem has a docs file. When you add a new rule, write a
research brief mirroring the existing structure (8-10 sections).
When you add a new predicate, follow the workstream_d_phase{n}.md
template. When you find a real-target bug, document the
chain-of-causation like `mod_abort_investigation.md`.

Every numerical value in the docs/ tree carries a DOI or source
file:line citation. This is Ivan's standing rule for the project
(memory: `feedback_macrocert_references`).
